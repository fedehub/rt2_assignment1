The action branch, should contain the same package in R\+OS, but with the {\bfseries{go\+\_\+to\+\_\+point node}} modelled as a {\bfseries{R\+OS action server}}, instead of a “simple” server. Given that, the robot Finite State Machine (F\+SM) node should now implement mechanisms for possibly cancelling the goal, when the related user command is received.

This branch is structured as reported below, in the following paragraphs.\hypertarget{md__r_e_a_d_m_e_autotoc_md4}{}\doxysubsection{Action folder}\label{md__r_e_a_d_m_e_autotoc_md4}
It contains the instruction for defining the expected {\bfseries{action}}.\hypertarget{md__r_e_a_d_m_e_autotoc_md5}{}\doxysubsection{Script folder}\label{md__r_e_a_d_m_e_autotoc_md5}
It contains\+:
\begin{DoxyEnumerate}
\item \href{https://github.com/fedehub/rt2_assignment1/blob/action/scripts/go_to_point.py}{\texttt{ go\+\_\+to\+\_\+point}}
\item \href{https://github.com/fedehub/rt2_assignment1/blob/action/scripts/user_interface.py}{\texttt{ user\+\_\+interface}}\+: If user enter 1, the command client calls the state machine with the start string (robot starts to move randomly). If it enters 0, in this branch we have defined a client for the action so that teh goal of the robot is canceled and the robot is stopped in the actual position through a twist message.
\end{DoxyEnumerate}\hypertarget{md__r_e_a_d_m_e_autotoc_md6}{}\doxysubsection{Launch folder}\label{md__r_e_a_d_m_e_autotoc_md6}

\begin{DoxyEnumerate}
\item \href{https://github.com/fedehub/rt2_assignment1/blob/action/srv/Command.srv}{\texttt{ Command.\+srv}}
\item \href{https://github.com/fedehub/rt2_assignment1/blob/action/srv/RandomPosition.srv}{\texttt{ Random\+Position.\+srv}}\+:
\begin{DoxyItemize}
\item As request it issues {\bfseries{two ranges}} of intervals for the {\bfseries{x}} and {\bfseries{y coordinates}}
\item As reply it provides a {\bfseries{random x,y}} and {\bfseries{theta}} {\bfseries{values}}
\end{DoxyItemize}
\end{DoxyEnumerate}\hypertarget{md__r_e_a_d_m_e_autotoc_md8}{}\doxysubsection{Src folder}\label{md__r_e_a_d_m_e_autotoc_md8}

\begin{DoxyEnumerate}
\item \href{https://github.com/fedehub/rt2_assignment1/blob/action/src/position_service.cpp}{\texttt{ postion\+\_\+service.\+cpp}}\+: It simply returns the random position which the robot has to reach.
\item \href{https://github.com/fedehub/rt2_assignment1/blob/action/src/state_machine.cpp}{\texttt{ state\+\_\+machine.\+cpp}}\+: By means of an user interface, the user is able of making the robot starts by entering the 1 integer value, the robot starts moving. There is one boolean value which becomes true and then
\begin{DoxyItemize}
